Agent roles

Team

RorkParity is not built to replace every agent. It is built to work alongside them. The main agent drives the roadmap. Supporting agents keep the spec tight, the build clean, and the checks honest.

FORGEMASTER

Plan

Reads the request and turns it into a buildable spec.

  • Chooses stack and template
  • Sets scope for v1
  • Names what gets deferred
SMITH

Build

Implements the spec without disturbing working parts.

  • Writes the page or feature
  • Patches only what changed
  • Keeps output static-first
WHETSTONE

Validate

Checks the result before it reaches the user.

  • Syntax and runtime risk
  • Scope creep checks
  • Minimal fix notes
LOREKEEPER

Curate

Keeps templates, docs, and reusable pieces aligned.

  • Template registry
  • Docs and manifests
  • Asset hygiene
CRIER

Publish

Packages release work. It does not ship on its own.

  • Deployment prep
  • SEO assets
  • Release packaging
MAIN + SUPPORT

Roadmap work

The main agent owns the roadmap. Supporting agents review, extend, and validate the work. That keeps the system collaborative instead of territorial.

  • One agent drafts
  • Another agent reviews
  • Roadmap stays visible
Roadmap

Current focus

  • Stabilize the builder: generation, preview, and iteration need to stay reliable.
  • Show the roles: keep Team visible from the main nav and landing page.
  • Stay game-first: keep Phaser and Three.js paths strong.
  • Keep persistence solid: saved projects, versions, and gallery manifest flow cleanly.
  • Defer mobile parity: native export waits until the web flow is stable.

Live ops timeline

Game-first. Static-first. Collaboration over replacement.
