Agent roles

Team Page

RorkParity is built as a multi-agent system. The site should show the roles, show the roadmap, and make it clear that the main agent works with supporting agents instead of pretending to do everything alone.

FORGEMASTER

Plan

Turns the user request into a tight spec. Chooses the template, stack, and scope. Cuts anything that does not ship in v1.

  • Reads the request
  • Sets scope
  • Flags deferred work plainly
SMITH

Build

Writes the game or feature from the spec. Keeps changes local and avoids regenerating working systems.

  • Generates files
  • Touches only the affected sections
  • Preserves working code
WHETSTONE

Validate

Checks for runtime risk, scope creep, and obvious breakage. Approves only when the result actually works.

  • Syntax and crash checks
  • Scope compliance
  • Minimal patch feedback
LOREKEEPER

Store

Manages templates, docs, and reusable assets. Keeps the library aligned to the current product direction.

  • Template registry
  • Docs and manifests
  • Asset hygiene
CRIER

Publish

Handles packaging and release steps. It packages. It does not ship without a human saying so.

  • Deploy prep
  • SEO assets
  • Release packaging only
COORDINATOR

Work with other agents

The primary agent works in parallel with supporting agents on roadmap and execution. That is the model.

  • One agent may draft the plan
  • Another may review or extend it
  • Roadmap stays visible to the user
